# 實驗 5：Allocator Massaging 完整模擬

## 目的

結合實驗 3（slab 結構觀察）與實驗 4（SLUBStick 時間側通道），
實際把一個 slab page 填滿攻擊者控制的 `msg_msg`，
重現論文 Section 5 的 Allocator Massaging 技術。

---

## 原理

攻擊者需要達到的目標狀態：

```
目標 slab page（整頁都是攻擊者的 msg_msg）

┌──────┬──────┬──────┬──────┬──────┬──────┬──────┐
│msg 0 │msg 1 │msg 2 │msg 3 │msg 4 │ ...  │msg 31│
└──────┴──────┴──────┴──────┴──────┴──────┴──────┘
  全部都是攻擊者控制的物件！
```

達成後，只要 TLB 側通道洩漏出 slab page 的基底位址，
每個物件的精確位址就可以直接計算：

```
msg_msg[n].addr = slab_base + 128 × n   （n = 0, 1, ..., 31）
```

---

## 三個步驟

### 步驟一：Drain（排空現有空位）

把 cache 現有的空位全部填滿假物件，
確保下一批分配必須從新的 slab page 開始：

```
排空前：cache 的空位混雜著其他 process 的物件殘留
排空後：所有舊空位都被假物件佔用
        → 下次分配 = 一定產生新的 slab page
```

### 步驟二：Massaging + SLUBStick

分配目標物件，同時用 `rdtsc` 量每次的時間：

```
慢分配 → 新 slab page 開始
快 快 快 ... 快（共 32 個）→ 這 32 個在同一個 slab page
慢分配 → 下一個 slab page 開始
快 快 快 ... 快（共 32 個）→ 又一個完整的 slab page
```

### 步驟三：驗證

統計每個 group 的物件數，確認等於 32。

```bash
gcc -O0 -o massaging massaging.c
./massaging
```

## Drain 數量的選擇

```
DRAIN_COUNT = 500（第一次嘗試，失敗）：
  → Drain 不夠，舊的空位沒清完
  → Group 1 有 224 個物件（混入了舊空位）

DRAIN_COUNT = 2000（修正後，成功）：
  → 確保所有舊空位都被填滿
  → Group 2 起每頁精確 32 個物件
```

實際攻擊中，drain 數量需要根據目標 cache 的大小動態決定。
論文使用更精細的方法，結合 SLUBStick 計時來確認 drain 是否完成。

---

## 閾值算法的選擇

| 算法 | 優點 | 缺點 | 適用場景 |
|------|------|------|---------|
| 最大間隔法 | 直覺、不需參數 | VM 噪音大時容易失效 | 安靜的環境 |
| 中位數 × N | 穩定、抗噪音 | 需要調整倍數 | VM 或噪音大的環境 |

本實驗使用**中位數 × 5**，在 VM 環境下表現穩定。

## 與論文的完整對應

```
論文 Section 5.1（D1 的 massaging）對應步驟：

① 分配大量假物件排空 page allocator 的低階 free list
   → 對應本實驗的 Drain 步驟

② 載入 kernel module，觸發 2MB → 4kB 頁面分割
   → 對應論文 D1 特有的步驟（本實驗未模擬）

③ 分配目標物件，佔用分割後的 4kB 頁面
   → 對應本實驗的 Target 分配步驟

④ 用 SLUBStick 確認整頁都是目標物件
   → 對應本實驗的計時 + 分組步驟
```

## 五個實驗的完整邏輯鏈

```
實驗 1  TLB Hit/Miss 有時間差（64 vs 489 cycles）
           ↓ 這個時間差讓攻擊者能「感知」kernel 的記憶體存取

實驗 2  TLB 容量有限（L2 STLB ≈ 2048 條目）
           ↓ 容量有限代表可以用 Evict 把目標條目擠出去

實驗 3  Slab Allocator 把同類物件放在同一頁
           ↓ 物件分布有規律，知道頁面位址就能算出每個物件位址

實驗 4  SLUBStick：用時間區分「新 slab page」
           ↓ 能識別哪些物件在同一頁

實驗 5  Allocator Massaging：把整頁填滿攻擊者的物件  ← 本實驗
           ↓
        整頁都是攻擊者的物件
        + TLB 側通道洩漏 slab page 基底位址
           ↓
        每個物件的精確位址 = slab_base + 128 × n
           ↓
        可靠的 kernel exploit（論文 Section 7）
```
